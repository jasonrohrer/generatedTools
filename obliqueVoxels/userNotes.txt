Both issues from the last round are fixed:

--Smoothness no longer live-updates the pixel preview:
  Root cause was the layer composite.  The pixel render shades the FLAT
  composite (union of visible layers), which is only rebuilt when g_flatDirty
  is set -- and that flag was set only by voxSet/voxErase (add/remove a voxel).
  Painting smoothness (Smoother tool) or Smooth/Unsmooth in the Selection
  panel mutate smoothFaces directly, so the composite kept the stale mask until
  some unrelated edit rebuilt it.  The smoothness *sliders* looked broken for
  the same reason -- the faces you'd just painted weren't in the composite yet,
  so the sliders had nothing to act on.  smoothOneFace() and selSmooth() now
  flag the composite dirty, so both painted smoothness and the sliders update
  the preview live again.  (Regression covered by OV_SELFTEST: smooth-composite.)

--Pencil tool now scribbles instead of rotating:
  Left-drag with the Pencil no longer orbits (right-drag still does).  It now
  works like the Scribble/Smoother tools: dragging paints a translucent ghost
  block onto every real surface the cursor sweeps over (each block sticks to the
  closest *real* voxel face, or the ground -- never ghost-on-ghost), and on
  mouse-up the whole set commits as ONE undo step.  A plain click still paints a
  single voxel (same code path).  As you noted, towers don't grow within a
  single drag (ghosts stick to real geometry only) -- scribble, release, and
  scribble again to build up.
